# Visão geral sobre Layout

### Layout

* **Construção de UI**

* **Jetpack compose**

  * Não está totalmente pronto para produção, entretanto já pode ser utilizado para desenvolvimento de apps de pequeno/médio porte

  * Trabalha com UI declarativo (construção do layout a partir de código em Kotlin)

* **Layout XML**

  * Ainda muito utilizado na construção de layouts

  * tipos de layout:

    * Parent

    * Componentes

    * Medidas de altura e largura

    * Personalização de atributos

* **ViewBinding** e **DataBinding**

  * Arquivo de *binding* gerado a partir de XML

  * Vantagem do em relação ao Java:  

    * Java: busca explícita (ID) por um componente

    * Kotlin: apenas referênciar um componente
