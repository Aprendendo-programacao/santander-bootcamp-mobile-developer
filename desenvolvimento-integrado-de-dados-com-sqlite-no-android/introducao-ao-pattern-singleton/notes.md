# Introdução a Pattern Singleton

### Pattern

* É um padrão de projeto que tem a finalidade de resolver/simplificar problemas

* **Exemplos**: Adapter, Singleton, Bridge, etc...

### Objeto estático

* Acessar os métodos e atributos de um objeto de forma global e fácil, além de não ter a necessidade de instanciá-la

### Pattern Singleton

* Objeto comum para toda a aplicação, ou seja, uma única instância de uma classe em toda a aplicação

* **Exemplo**

  ```kotlin
  object Singleton {
    // Código
  }
  ```
